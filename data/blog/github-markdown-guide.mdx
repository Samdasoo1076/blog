# 설명

Markdown 및 Mdx 구문 분석은 **통합** 및 기타 비고 및 리프레쉬 패키지를 통해 지원됩니다. 'next-mdx-remote'를 사용하면 webpack을 건드리지 않고도 '.mdx' 및 '.md' 파일을 보다 유연한 방식으로 구문 분석할 수 있습니다.

GitHub 마크다운이 사용됩니다. 'mdx-prism'은 코드 블록에 대한 구문 강조 표시 기능을 제공합니다. 다음 소스 코드로 확인해 봅시다.

다음 마크다운 치트 시트는 다음에서 각색되었습니다.: https://guides.github.com/features/mastering-markdown/

# 마크다운이란?

Markdown은 웹에서 텍스트의 스타일을 지정하는 방법입니다. 문서의 표시를 제어합니다. 단어의 서식을 굵게 또는 기울임꼴로 지정하고, 이미지를 추가하고, 목록을 만드는 것은 Markdown으로 할 수 있는 작업 중 일부에 불과합니다. 대부분 Markdown은 '#' 또는 '\*'와 같이 알파벳이 아닌 몇 가지 문자가 포함된 일반 텍스트입니다.

# 구문 가이드

다음은 GitHub.com 또는 텍스트 파일의 어디에서나 사용할 수 있는 Markdown 구문에 대한 개요입니다.

## 헤더 태그

```
# 이것은 h1 태그입니다.

## 이것은 h2 태그입니다.

#### 이것은 h4 태그입니다.
```

# 이것은 h1 태그입니다.

## 이것은 h2 태그입니다.

#### 이것은 h4 태그입니다.

## 강조 태그

```
_이것은 기울임꼴 텍스트 입니다._

**이것은 굵은꼴 텍스트 입니다.**

_이것은 **강조** 텍스트 입니다._
```

_이것은 기울임꼴 텍스트 입니다._

**이것은 굵은꼴 텍스트 입니다.**

_이것은 **강조** 텍스트 입니다._

## 구문 강조

다음은 구문 강조 표시를 사용하는 방법의 예입니다. [GitHub Flavored Markdown](https://help.github.com/articles/basic-writing-and-formatting-syntax/):

````
```js:fancyAlert.js
function fancyAlert(arg) {
  if (arg) {
    $.facebox({ div: '#foo' })
  }
}
```
````

스타일이 지정된 코드 제목으로 멋지게 채색된 코드 입니다.

```js:fancyAlert.js
function fancyAlert(arg) {
  if (arg) {
    $.facebox({ div: '#foo' })
  }
}
```


## 체크 리스트

```
- [x] 목록 구문 필요(정렬되지 않은 목록 또는 정렬된 목록 지원)
- [x] 체크 박스에 체크 표시가 되어 있다면 완료된 표시입니다.
- [ ] 체크 되지 않은 비완료된 리스트
```

- [x] 목록 구문 필요(정렬되지 않은 목록 또는 정렬된 목록 지원)
- [x] 체크 박스에 체크 표시가 되어 있다면 완료된 표시입니다.
- [ ] 체크 되지 않은 비완료된 리스트

## 테이블

단어 목록을 어셈블하고 하이픈 '-'(첫 번째 행의 경우)으로 나눈 다음 각 열을 파이프 '|'로 구분하여 테이블을 만들 수 있습니다.

```
| 첫번째 머릿글                | 두번째 머리글                 |
| --------------------------- | ---------------------------- |
| 첫번째 쉘                    | 두번째 쉘                    |
| 컨텐츠 안에 첫번째 컬럼       | 컨텐츠 안에 두번째 컬럼        |
```

| 첫번째 머릿글           | 두번째 머리글           |
| ----------------------- | ----------------------- |
| 첫번째 쉘               | 두번째 쉘               |
| 컨텐츠 안에 첫번째 컬럼 | 컨텐츠 안에 두번째 컬럼 |

## 취소선

두 개의 물결표로 줄 바꿈된 단어(예: '~~this~~')는 ~~줄이 그어진 ~~로 나타납니다.
